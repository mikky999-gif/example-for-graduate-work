# Платформа по перепродаже вещей «SkyPro»

Бэкенд-часть дипломного проекта на базе **Spring Boot**. Реализует API для работы классифайда: регистрация пользователей, управление объявлениями и комментариями, а также безопасная загрузка изображений.

## 🛠 Стек технологий

*   **Язык:** Java 13
*   **Сборка:** Maven
*   **Фреймворк:** Spring Boot 2.7.15
*   **Безопасность:** Spring Security (Basic Auth)
*   **Работа с БД:** Spring Data JPA, Hibernate, PostgreSQL
*   **Миграции:** Liquibase
*   **Валидация:** Bean Validation (Hibernate Validator)
*   **Объектно-реляционное маппирование:** MapStruct
*   **Документация:** Springdoc OpenAPI UI (Swagger)
*   **Логирование:** SLF4J + Logback
*   **Пул соединений:** HikariCP

## 📋 Описание функционала

Проект реализует RESTful API со следующим набором возможностей:

### Аутентификация и авторизация
*   Регистрация новых пользователей (`/register`) с ролью `USER` или `ADMIN`.
*   Вход через Basic-аутентификацию (логин и пароль передаются в заголовке `Authorization`).
*   Разграничение прав: обычные пользователи работают только со своими данными, администраторы имеют полный доступ ко всем ресурсам.

### Объявления (Ads)
*   Получение списка всех объявлений с пагинацией (`GET /ads`).
*   Просмотр детальной информации об объявлении (`GET /ads/{id}`).
*   Создание нового объявления с картинкой (`POST /ads`).
*   Редактирование и удаление *своих* объявлений (`PATCH`, `DELETE /ads/{id}`).
*   Администратор может редактировать и удалять любые объявления.

### Комментарии (Comments)
*   Оставление комментариев под объявлениями (`POST /ads/{adId}/comments`).
*   Редактирование и удаление *своих* комментариев.
*   Администратор может модерировать все комментарии.

### Профиль пользователя
*   Просмотр и обновление данных профиля (`GET/PATCH /users/me`).
*   Смена пароля (`POST /users/set_password`).
*   Загрузка аватара пользователя (`PATCH /users/me/image`).

### Работа с изображениями
*   Картинки объявлений и аватары сохраняются в файловую систему сервера (директория `uploads/`).
*   Доступ к файлам предоставляется через отдельный контроллер `/images/{filename}`.

## 🚀 Запуск проекта локально

Для запуска приложения вам потребуется установленный **Java 13** и запущенный сервер **PostgreSQL**.

1.  **Настройка базы данных**
    Создайте базу данных в PostgreSQL:
    ```sql
    CREATE DATABASE example-for-graduate-work;
    ```
    *(Параметры подключения уже указаны в application.properties)*

2.  **Конфигурация**
    Скопируйте настройки из файла `src/main/resources/application.properties`. Убедитесь, что путь `app.images.root-path` существует на вашем диске.

3.  **Запуск**
    Соберите и запустите приложение:
    ```bash
    ./mvnw clean install
    ./mvnw spring-boot:run
    ```
    Приложение будет доступно по адресу: `http://localhost:8080`

4.  **Документация API**
    После запуска откройте Swagger UI в браузере:
    [http://localhost:8080/swagger-ui/index.html](http://localhost:localhost:8080/swagger-ui/index.html)
